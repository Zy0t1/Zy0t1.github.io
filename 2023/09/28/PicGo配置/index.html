图床配置

更新于 2023-09-28 阅读次数： 本文字数： 1k 阅读时长 ≈ 2 分钟

摘要：感觉blog有时还是有图片才能说清楚。配置采用ImgTP+PicGo+Typora

PicGo+Typora

安装PicGo

首先需要保证有安装node.js下载 | Node.js 中文网 (nodejs.cn)，且版本>=16。官网PicGo

进入下载安装界面，最新的测试版版（2.4）的能够在github中找到，其余的都是正式发行版（2.3.1）

安装正式版的即可，没有多大的区别

这里以山东大学镜像站为例子，进入其右侧链接

之后点击v2.3.1即可下载，安装。具体安装过程不再演示

配置PicGo

PicGo的作用仅相当于快速帮助我们将图片上传到图床中，所以我们还需要配置图床。下面是其默认支持的图床

本人选择的是imgtp（穷啊）ImgTP - 免费公共图床

当然还有其他许多图床以插件的形式支持，具体请见GitHub - PicGo/Awesome-PicGo: A collection of awesome projects using PicGo.

如果选择的图床并非上述默认支持，可以在PicGo的 “插件设置” 中进行搜索，之后安装。上述网站中的插件大部分可以用这种方式，少部分需要手动安装，这里不过多叙述

在图床注册号帐号之后，在设置里找到Token项，copy一下

之后进入到PicGo中，插件选项，配置uploader-imgtp

按照框子进行输入（该图床插件并不需要Token，但是大多都是必须的）

确认之后，记得到PicGO的图床设置中，将你的图床设置为默认图床

Question

  1. 端口设置，一般默认就好，配置端口36677（位置在”PicGo设置”——>”设置Server”）

​ 需要与Typora的端口设置对应，也就是上面验证时的命令：

​ using http://127.0.0.1:36677/upload

  1. 后端Token获取失败，需要在配置时，将”上传模式”更改为Front-End

  1. 有时显示该图片被阻塞，大概率是墙的问题。就比如SM.MS图床，就需要设置它的备用域名

注意：大多图床都会检测VPN，如果是开着梯子上传的，则会上传失败
